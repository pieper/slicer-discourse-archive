## Post #1 by @jcfr (2020-03-30 22:21 UTC)

<p>As discussed in <a href="https://github.com/Slicer/Slicer/pull/4783#issuecomment-605662343">this issue</a>, the <code>DOC:</code> prefix is now valid when writing commit message.</p>
<p>The <a href="https://www.slicer.org/wiki/Documentation/Nightly/Developers/Style_Guide#Commits">Commit Style Guide</a> and the <a href="https://www.commitcheck.com">commit check</a> have both been updated.</p>
<p>For reference,  here is the regex used to validate the commit title: <code>(BUG|COMP|DOC|ENH|PERF|STYLE)\: \w+</code></p>
