## Post #3 by @lassoan (2018-11-08 14:04 UTC)

<p>Slicer reconstructs models from volumes very robustly and accurately. DICOM image import is the only part of the workflow that might go wrong in case the scanner saves images that are not fully compliant to the DICOM standard.</p>
<ul>
<li>What CT scanner brand/model do you use?</li>
<li>What image resolution and slice spacing was set in the scanner?</li>
<li>What image spacing values do you see in Volumes module / Volume Information / Image Spacing?</li>
<li>Did you use DICOM module to load the image? Did you get any warning when you loaded the image?</li>
<li>What Slicer version did you use?</li>
</ul>

## Post #5 by @lassoan (2018-11-08 18:28 UTC)

<p>You can find CT scanner manufacturer/model in DICOM fields: in DICOM module, select your image series and click Metadata button (look for Manufacturer and ManufacturerModelName fields). It is important to know this because clinical scanners implement DICOM standard correctly, but many microCTs, industrial, or pre-clinical devices have faulty implementation.</p>
<p>Please use at least Slicer-4.10.0.</p>
