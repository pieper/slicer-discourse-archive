## Post #13 by @pieper (2018-10-25 02:00 UTC)

<p>Those files look like old GE file (Genesis, which was used a lot in Slicer2 and somewhat in Slicer3, but not used much for quite a while).  As I recall the ITK readers for these may have been turned on or off multiple times because there was a lot of debugging of the dynamic library loading methods for image IO code.</p>
<p>If it’s workable for you, I’d suggest converting the old data into a format like nrrd or nii.gz to save space and be more compatible.</p>

ITK’s IO constructs volume from files in a folder automatically, having any single file, may this approach confuses Slicer somehow.<br>
