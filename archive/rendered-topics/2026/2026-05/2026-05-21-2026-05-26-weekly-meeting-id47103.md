## Post #4 by @ebrahim (2026-05-26 17:15 UTC)

<p>Meeting notes:</p>
<h2><a name="p-133977-slicer-512-everyone-1" class="anchor" href="#p-133977-slicer-512-everyone-1" aria-label="Heading link"></a>Slicer 5.12 (Everyone)</h2>
<p>Slicer 5.12 release issue now exists: <a href="https://github.com/Slicer/Slicer/issues/9180" class="inline-onebox">Release Slicer v5.12 · Issue #9180 · Slicer/Slicer · GitHub</a></p>
<p>It is a good time to release 5.12 soon because we have a fairly clean dashboard and we are about tackle two problems that will create a broken nightly for a while: transtion to Qt6 and support mac arm64.</p>
<h2><a name="p-133977-imported-symbols-in-python-console-steve-ebrahim-2" class="anchor" href="#p-133977-imported-symbols-in-python-console-steve-ebrahim-2" aria-label="Heading link"></a>Imported symbols in python console (Steve, Ebrahim)</h2>
<p>We should restore the legacy behavior in terms of things in <code>slicer.util</code> that were previously automatically imported in the slicer python console. Import everything, or almost everything, from slicer.util explicitly <a href="https://github.com/Slicer/Slicer/blob/208adb862bc329e6370ae400fef1b18e9ac54cbd/Base/Python/slicerqt.py#L14-L20">here</a>, or do a wildcard import but explicitly control <code>__all__</code> for <code>slicer.util</code>.<br>
Opened issue: <a href="https://github.com/Slicer/Slicer/issues/9182" class="inline-onebox">Restore slicer.util imports in python console · Issue #9182 · Slicer/Slicer · GitHub</a></p>
<h2><a name="p-133977-support-for-qtqthread-in-3d-slicer-python-thibault-3" class="anchor" href="#p-133977-support-for-qtqthread-in-3d-slicer-python-thibault-3" aria-label="Heading link"></a>Support for qt.QThread in 3D Slicer python (Thibault)</h2>
<p>We have started looking into the support of qt.QThread in 3D python.<br>
Currently running qt.QThreads would either result in running in the main thread or the application crashing.</p>
<p>As we need to add GIL locking and do proper non regression testing, threading support should be added comming Slicer 6.0</p>
<h2><a name="p-133977-to-integrate-slicerlayerdm-into-slicer-core-thibault-4" class="anchor" href="#p-133977-to-integrate-slicerlayerdm-into-slicer-core-thibault-4" aria-label="Heading link"></a>To integrate SlicerLayerDM into Slicer core (Thibault)</h2>
<p>The SlicerLayerDM extension has proven to be very useful to implement and integrate advanced Slicer widgets in both Python and C++.</p>
<p>We plan to integrate the SlicerLayerDM in the core of Slicer starting preview release Slicer 5.13+.</p>
