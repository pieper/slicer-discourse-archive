<p>Mauro adding to Slicer/Utilities in a PR</p>
<p>After review and testing and agreement, we can make <a href="http://slicer.org/install.sh">slicer.org/install.sh</a> redirect to it on github.</p>
<p>Still need someone to test on macOS:</p>
<pre><code class="lang-auto">curl -fsSL https://raw.githubusercontent.com/mauigna06/slicer-installer/main/install.sh | sh
</code></pre>
<p>And here is how to try it out on windows right now:</p>
<pre><code class="lang-auto">powershell -c "irm https://raw.githubusercontent.com/mauigna06/slicer-installer/main/install.ps1 | iex"
</code></pre>
<p>Consider:</p>
<ul>
<li>a no-interaction mode, for people who want to use the script in automations</li>
<li>an automatic dependency installation mode, for people who want to use the script to create docker images</li>
<li>the possibility of packaging an installer script into the tarball, in the linux case</li>
<li>representing system dependencies in a separate file that the script references and that the docs can also reference to report system dependencies, so that they are in one place and never go out of sync</li>
</ul>
