## Post #3 by @ebrahim (2026-03-03 15:47 UTC)

<p>Notes from the meeting:</p>
<h2><a name="p-132412-recent-slicerorg-instability-1" class="anchor" href="#p-132412-recent-slicerorg-instability-1" aria-label="Heading link"></a>Recent <a href="http://slicer.org">slicer.org</a> instability</h2>
<p>It is hosted on digitalocean VM, not at Kitware, unlike most other hosted services. For example <a href="http://download.slicer.org">download.slicer.org</a> is netflify. We should move <a href="http://slicer.org">slicer.org</a> to a netlify page; digitalocean vm is not very accessible and for changes it is pulling from a repo anyway.</p>
<p>The <a href="http://slicer.org">slicer.org</a> domain expires in 2027 so we should probably pin down who owns and manages it.</p>
<p>Next project week meeting will be in Boston so that will be a good chance to address some of this <a href="http://slicer.org">slicer.org</a> stuff – get it to netlify and potentially hosted at Kitware.</p>
<p>We can also bring into project week the challenging project of moving factoies to github actions. Paid runners are one option, but it depends on how much this costs. A lower cost but higher maintenance middle-ground is self-hosted runners at Kitware that maintainers can access (perhaps in a “DMZ”).</p>
