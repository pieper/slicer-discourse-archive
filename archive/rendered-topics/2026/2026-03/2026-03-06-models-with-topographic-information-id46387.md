## Post #4 by @drnoorfatima (2026-03-06 10:15 UTC)

<p>Exactly right. To add a bit more precision:</p>
<p>The colors represent signed surface distance between two meshes, a specimen and a reference. Specifically:<br>
Red/yellow = this surface point is raised relative to the reference (taller cusp, more relief)<br>
Blue = this surface point is sunken relative to the reference (flatter, worn down, or evolutionarily reduced)<br>
Green= near zero difference, both surfaces match at that point</p>
<p>The word “signed” matters here. It tells you not just HOW DIFFERENT two surfaces are, but in WHICH DIRECTION.</p>

## Post #5 by @drnoorfatima (2026-03-06 10:17 UTC)

<p>I built a fully automated pipeline for generating surface distance maps (like the color deviation maps in Schultz &amp; Martin 2014) entirely inside 3D Slicer, no Blender required. It uses <code>vtkDistancePolyDataFilter</code> with signed distances, ColdToHotRainbow color mapping, and exports publication-quality PNG directly from the Python console.</p>
<p>Tested and working on Slicer 5.7. Distance range, color scale bar, teal wireframe reference overlay, all automated in one script.</p>
<p>If you have specimens you want me to help you with, feel free to DM me. Happy to run it for you.</p>
