## Post #3 by @DeepaKrishnaswamy (2025-06-03 15:34 UTC)

<p>Proposal: Creating streamlined workflows in Slicer<br>
Annotating large datasets in a streamlined and efficient manner is difficult. In Slicer, multiple extensions have been developed to improve workflow issues, including <a href="https://github.com/JoostJM/SlicerCaseIterator" rel="noopener nofollow ugc">CaseIterator</a>, <a href="https://github.com/zapaishchykova/SegmentationReview" rel="noopener nofollow ugc">SegmentationReview</a>, <a href="https://github.com/cpinter/SlicerSegmentationVerification" rel="noopener nofollow ugc">SlicerSegmentationVerification</a>, <a href="https://github.com/neuropoly/slicercart" rel="noopener nofollow ugc">SlicerCART</a>, <a href="https://github.com/ungi/SlicerUltrasound" rel="noopener nofollow ugc">SlicerUltrasound</a> and <a href="https://github.com/SlicerProstate/mpReview" rel="noopener nofollow ugc">mpReview</a>. Possible topics of discussion include:</p>
<ul>
<li>best practices</li>
<li>what these extensions have in common</li>
<li>how to create the infrastructure to support the annotation workflow</li>
<li>involvement of other packages like <a href="https://github.com/Project-MONAI/MONAILabel" rel="noopener nofollow ugc">MONAILabel</a></li>
<li>tasks that a workflow should contain - curation, annotation, review, comparing multiple annotations, classification</li>
</ul>

## Post #6 by @aylward (2025-06-04 13:10 UTC)

<p>Dynamic physiology for human digital twins. Focus: using slicer to go from a static CT scan of a patient to a digital twin model that has respiratory and cardiac motion.  Ultimately will incorporate more data from other modalities for more precise patient specific motion estimations and simulations of blood flow and estimation of forces.  Useful for minimally invasive surgical planning, guidance, and outcome prediction. Goal of breakout: learn of ongoing work and build community.</p>
