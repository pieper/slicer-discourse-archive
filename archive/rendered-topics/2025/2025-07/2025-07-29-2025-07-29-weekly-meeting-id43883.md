<p>To eliminate the need to interact with the Qt layer directly, <a class="mention" href="/u/thibault_pelletier">@Thibault_Pelletier</a> proposed enhancing <code>vtkMRMLAbstractViewNode</code> with new properties and mechanisms:</p>

<li>
<p><strong>New weak pointers</strong>:</p>
<ul>
<li><code>DisplayableManagerGroup</code></li>
<li><code>RenderWindow</code></li>
<li><code>Logic</code> (for accessing slice/view logic)</li>
</ul>
</li>
<li>
<p><strong>New events</strong>:</p>
<ul>

<li><code>ForceRender</code></li>

  auto* crosshairDisplayableManager =

</code></pre>

<h3><a name="p-127332-proposed-api-vtkmrmlscenegetnodesbyclass-8" class="anchor" href="#p-127332-proposed-api-vtkmrmlscenegetnodesbyclass-8" aria-label="Heading link"></a>Proposed API: <code>vtkMRMLScene::GetNodesByClass</code></h3>
<p><a class="mention" href="/u/thibault_pelletier">@Thibault_Pelletier</a> suggested adding a utility method to retrieve all nodes of a given type with proper type casting:</p>
<pre data-code-wrap="cpp"><code class="lang-cpp">template &lt;class T&gt;
std::vector&lt;T*&gt; GetNodesByClass(const char* className)
{
  std::vector&lt;vtkMRMLNode*&gt; nodes;
  GetNodesByClass(className, nodes);

  std::vector&lt;T*&gt; typedNodes;
  for (const auto&amp; node : nodes)
  {
    auto castNode = T::SafeDownCast(node);
    if (castNode)
    {
      typedNodes.emplace_back(castNode);
    }
  }
  return typedNodes;
}
</code></pre>
