<p><a class="mention" href="/u/pieper">@pieper</a> According to the CMake documentation for <a href="https://cmake.org/cmake/help/latest/module/ExternalProject.html#git" rel="noopener nofollow ugc">ExternalProject git usage</a>, shallow depth cloning is not supported as we use commit hashes for our external projects. See relevant CMake documentation snippets below:</p>
<blockquote>
<ul>
<li>Using a specific git hash ensures that the main project’s own history is fully traceable to a specific point in the external project’s evolution. If a branch or tag name is used instead, then checking out a specific commit of the main project doesn’t necessarily pin the whole build to a specific point in the life of the external project. The lack of such deterministic behavior makes the main project lose traceability and repeatability.</li>
</ul>
</blockquote>
<blockquote>
<p>If <code>GIT_SHALLOW</code> is enabled then <code>GIT_TAG</code> works only with branch names and tags. A commit hash is not allowed.</p>
</blockquote>
<blockquote>
<p><code>GIT_SHALLOW &lt;bool&gt;</code></p>
<p><em>Added in version 3.6.</em></p>
<p>When this option is enabled, the <code>git clone</code> operation will be given the <code>--depth 1</code> option. This performs a shallow clone, which avoids downloading the whole history and instead retrieves just the commit denoted by the <code>GIT_TAG</code> option.</p>
</blockquote>
