## Post #4 by @jcfr (2025-10-21 13:35 UTC)

<h3><a name="p-129362-meeting-notes-1" class="anchor" href="#p-129362-meeting-notes-1" aria-label="Heading link"></a>Meeting Notes</h3>
<h4><a name="p-129362-dicom-documentation-2" class="anchor" href="#p-129362-dicom-documentation-2" aria-label="Heading link"></a>DICOM Documentation</h4>
<ul>
<li>Reviewed <a href="https://github.com/Slicer/Slicer/pull/8786">PR-8786</a>, which provides documentation on <strong>Slicer integration in hospital/PACS environments</strong>.</li>
<li>The goal is to offer practical guidance for connecting Slicer or Slicer-based applications to DICOM AEs in clinical workflows.</li>
<li>Documentation reflects assumptions like the inability to auto-discover DICOM AEs, the need for pre-configured endpoints, and support for both <strong>push</strong> (C-STORE) and <strong>pull</strong> (C-FIND/C-MOVE) operations.</li>
</ul>
<h4><a name="p-129362-visual-dicom-browser-3" class="anchor" href="#p-129362-visual-dicom-browser-3" aria-label="Heading link"></a>Visual DICOM Browser</h4>
<p>Reviewed project from <a href="https://projectweek.na-mic.org/PW42_2025_GranCanaria/Projects/VisualDICOMBrowser/">42nd NA-MIC Project Week: Visual DICOM Browser</a>.</p>
<p>Key Features:</p>
<ul>
<li>Proxying of DICOM requests (to route queries through another service).</li>
<li>Multi-threaded querying for improved responsiveness.</li>
</ul>
<p>Discussion Highlights:</p>
<ul>
<li>
<p>Clear distinction between research and clinical workflows: clinical usage is typically <strong>patient-centric</strong>, with a known patient ID.</p>
</li>
<li>
<p>Encouraged <strong>refactoring the existing DICOM browser</strong> to leverage the same model used by the Visual DICOM browser.</p>
</li>
<li>
<p>Considering refactoring of the backend to support <strong>TLS-based querying</strong>.</p>
</li>
<li>
<p>Usability suggestions:</p>
<ul>
<li>Add a <strong>more visible button</strong> to toggle the Visual Browser (currently hard to find).</li>
<li>Avoid displaying placeholder thumbnails when none are available, to improve user experience.</li>
</ul>
</li>
</ul>
<p>Next Steps and Roadmap:</p>
<ul>
<li>See <a href="https://github.com/commontk/CTK/issues/1230">commontk/CTK#1230</a> and associated <a href="https://github.com/commontk/CTK/issues/1230#issuecomment-3402330952">comment notes</a> for the 2025 roadmap for Visual DICOM.</li>
</ul>
