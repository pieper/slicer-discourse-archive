<h4><a name="p-127465-symbol-loading-issues-on-linux-6" class="anchor" href="#p-127465-symbol-loading-issues-on-linux-6" aria-label="Heading link"></a>Symbol Loading Issues on Linux</h4>
<ul>
<li>
<p>Investigating GSSAPI symbol conflicts involving <code>libtirpc</code> on modern Linux systems (e.g., Ubuntu 25.04).<br>
See <a href="https://github.com/Slicer/Slicer/issues/8626">#8626</a> for the top-level tracking issue.</p>
</li>
<li>
<p>The issue arises when Slicer is launched and <code>libtirpc.so.3</code> attempts to resolve versioned GSSAPI symbols that are missing from Slicer’s bundled stub (<code>libgssapi_krb5.so</code>), resulting in an error such as:</p>
<pre><code class="lang-auto">undefined symbol: GSS_C_NT_USER_NAME, version gssapi_krb5_2_MIT
</code></pre>
</li>
<li>
<p>The stubbed GSSAPI library was originally introduced to avoid OpenSSL version conflicts between the system GSSAPI and Slicer’s internally built OpenSSL.<br>
See <a href="https://github.com/Slicer/Slicer/pull/8598">PR #8598</a> for background.</p>
</li>
<li>
<p>Current efforts are focused on:</p>
<ul>
<li>Identifying what introduces the <code>libtirpc</code> dependency (e.g., Qt, ITK, Python, or transitive system libraries). See <a href="https://github.com/Slicer/Slicer/issues/8626">#8626</a></li>
<li>Evaluating whether the <code>krb5-gssapi-stub</code> should be improved or removed</li>
<li>Avoiding the need for developers or users to build Qt from source unless absolutely necessary</li>
</ul>
</li>
<li>
<p>Related discussion: See <a href="https://discourse.slicer.org/t/slicer-build-environment-upgraded-to-qt5-almalinux8-gcc14/43802/4">this post</a> by <a class="mention" href="/u/davide_punzo">@Davide_Punzo</a></p>
</li>
<li>
<p>Documentation update:<br>
A new section will be added to the <a href="https://slicer.readthedocs.io/en/latest/developer_guide/build_instructions/linux.html">Linux build instructions</a> covering known issues and workarounds specific to Ubuntu 25.04.<br>
See <a href="https://github.com/Slicer/Slicer/issues/8627">#8627</a></p>
</li>
</ul>
<h4><a name="p-127465-qt5-version-update-from-5152-to-51517-7" class="anchor" href="#p-127465-qt5-version-update-from-5152-to-51517-7" aria-label="Heading link"></a>Qt5 Version Update from <code>5.15.2</code> to <code>5.15.17</code></h4>
