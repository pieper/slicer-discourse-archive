## Post #2 by @jcfr (2025-08-19 13:18 UTC)

<h3><a name="p-127734-meeting-notes-1" class="anchor" href="#p-127734-meeting-notes-1" aria-label="Heading link"></a>Meeting Notes</h3>
<h4><a name="p-127734-slicerwarp-experiments-2" class="anchor" href="#p-127734-slicerwarp-experiments-2" aria-label="Heading link"></a>SlicerWarp Experiments</h4>
<p><a class="mention" href="/u/pieper">@pieper</a> presented an early integration of <a href="https://github.com/NVIDIA/warp">NVIDIA Warp</a> into 3D Slicer. Warp is a Python/CUDA-based simulation framework that supports CPU and GPU execution and integrates efficiently with PyTorch. Notable features include:</p>
<ul>
<li>All simulations are differentiable by default.</li>
<li>Integration with Slicer is performed using NumPy arrays.</li>
<li>Warp is being refactored into a standalone library called <a href="https://newton-physics.github.io/newton/guide/overview.html">Newton Physics</a>, which will continue using Warp under the hood.</li>
</ul>
<p><strong>Licensing</strong><br>
Warp was recently relicensed under the permissive Apache 2.0 license, making it more compatible with open-source projects like Slicer.</p>
<p><strong>Complement to SOFA</strong><br>
The group discussed how <code>SlicerWarp</code> could serve as a lightweight and performant alternative or complement to SOFA, particularly since many of the integration patterns with Slicer are similar.</p>

<h4><a name="p-127734-whisper-slicer-integration-experiment-3" class="anchor" href="#p-127734-whisper-slicer-integration-experiment-3" aria-label="Heading link"></a>Whisper &amp; Slicer Integration Experiment</h4>
<p>Prototype by <a class="mention" href="/u/mau_igna_06">@mau_igna_06</a>:</p>
<p>A live demo was shown that enables voice-based interaction with Slicer through a web browser. The system:</p>
<ul>
<li>Uses a WebAssembly (WASM)-compiled version of <a href="https://github.com/ggml-org/whisper.cpp/tree/master/examples/command.wasm">whisper.cpp</a> for speech-to-text processing.</li>
<li>Runs entirely in the browser and sends parsed voice commands to Slicer using the existing <strong>WebServer</strong> module.</li>
</ul>
<p>This proof-of-concept highlights a promising direction for making Slicer more accessible via natural language interfaces.</p>
