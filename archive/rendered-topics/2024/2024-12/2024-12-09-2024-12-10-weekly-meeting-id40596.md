<h4><a name="p-120362-curved-planar-reformation-cpr-5" class="anchor" href="#p-120362-curved-planar-reformation-cpr-5" aria-label="Heading link"></a>Curved Planar Reformation (CPR)</h4>
<ul>
<li>In the context of <a href="https://github.com/Slicer/Slicer/issues/7179">Slicer Issue #7179</a>, work is underway to integrate capabilities from <a href="https://github.com/PerkLab/SlicerSandbox#curved-planar-reformat">SlicerSandbox’s CPR implementation</a>.
<ul>
<li>The current SlicerSandbox approach relies on the <code>ResampleScalarVectorDWIVolume</code> CLI.</li>
<li>We discussed adding an API to support registering “resampler” in the Slice logic class implemented in <code>Libs/MRML/Logic</code>.</li>
<li><strong>Future Goals</strong>: Develop support for various use cases, including:
<ul>
<li>Data fitting in GPU memory.</li>
<li>Data fitting only in RAM.</li>
<li>Data that does not fit in memory.</li>
</ul>
</li>
<li><strong>Initial Focus</strong>: Register a resampler using the existing <code>ResampleScalarVectorDWIVolume</code> CLI.</li>
</ul>
</li>
</ul>
