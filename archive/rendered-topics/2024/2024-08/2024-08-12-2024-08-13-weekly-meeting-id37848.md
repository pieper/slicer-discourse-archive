## Post #3 by @jcfr (2024-08-15 14:33 UTC)

<h3><a name="p-115203-meeting-notes-1" class="anchor" href="#p-115203-meeting-notes-1" aria-label="Heading link"></a>Meeting Notes</h3>
<h4><a name="p-115203-sofa-2" class="anchor" href="#p-115203-sofa-2" aria-label="Heading link"></a>SOFA</h4>
<ul>
<li>
<p><strong>LapGym Overview</strong>: LapGym is an open-source framework designed for Reinforcement Learning (RL) in robot-assisted laparoscopic surgery. More information can be found at the official <a href="https://www.sofa-framework.org/applications/plugins/lapgym/">SOFA Framework website</a>.</p>
</li>
<li>
<p><strong>Paul’s Post-Doc</strong>: Paul is set to begin a post-doctoral position at Johns Hopkins, which could lead to further research and collaboration opportunities.</p>
</li>
<li>
<p><strong>Potential Applications</strong>: There are potential applications for SlicerHeart in conjunction with SOFA and LapGym, which should be explored further.</p>
</li>
</ul>
<h4><a name="p-115203-nvapi-integration-3" class="anchor" href="#p-115203-nvapi-integration-3" aria-label="Heading link"></a>NVAPI Integration</h4>
<ul>
<li>The discussion focused on building NVAPI within the Slicer environment. Relevant resources include the <a href="https://discourse.slicer.org/t/how-to-build-nvidia-nvapi-in-slicer-environment/37771">Slicer Discourse thread</a> and the official <a href="https://developer.nvidia.com/gameworksdownload#?search=nvapi">NVIDIA Developer page</a>.</li>
</ul>
<h4><a name="p-115203-morphodepot-4" class="anchor" href="#p-115203-morphodepot-4" aria-label="Heading link"></a>MorphoDepot</h4>
<ul>
<li><strong>GitHub Integration</strong>: We discussed how to map GitHub operations to the collaborative segmentation process within MorphoDepot. Existing GitHub credentials support, particularly the <a href="https://github.com/Slicer/Slicer/blob/55563bc1ec41acb7cbb9e48543e93f364fcd9cba/Utilities/Scripts/SlicerWizard/GithubHelper.py#L61">SlicerWizard’s GithubHelper.py</a>, may be reused for this purpose.</li>
</ul>
<h4><a name="p-115203-new-icons-proposal-5" class="anchor" href="#p-115203-new-icons-proposal-5" aria-label="Heading link"></a>New Icons Proposal</h4>
<ul>
<li>
<p><strong>Slice Viewer Icon Review</strong>: The proposal for new slice viewer icons was reviewed, as detailed in the <a href="https://discourse.slicer.org/t/2024-08-13-weekly-meeting/37848/2">Slicer Discourse thread</a>. Key points include:</p>
<ul>
<li>
<p><strong>Concept Clarification</strong>: A clear differentiation between “view” and “volume” was discussed, with the suggestion to consistently represent “view” using the color green.</p>
</li>
<li>
<p><strong>Label Map Visibility</strong>: A proposal was made to hide the label map row if no label map is loaded in the scene.</p>
</li>
<li>
<p><strong>Blending Options</strong>: To better support the visualization of non-overlapping volumes, making blending optional was suggested.</p>
</li>
</ul>
</li>
</ul>
