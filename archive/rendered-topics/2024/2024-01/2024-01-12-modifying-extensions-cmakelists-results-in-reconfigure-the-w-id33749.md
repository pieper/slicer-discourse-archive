## Post #2 by @lassoan (2024-01-15 05:44 UTC)

<p>If you bundle an extension in a custom Slicer application then the extension is built as part of the application and so modification of the CMakeLists.txt file triggers reconfiguration of the entire applicaiton. This is indeed a quite long process - may take 5-10 minutes on Windows on a computer with a slow disk, using the MSVC generator.</p>
<p>However, since modification of CMakeLists.txt files is a very rare event (you add a source file once and then modify it hundreds or thousands of times) the task of reducing this configuration time would have low priority compared to most fixes and development of new features.</p>
