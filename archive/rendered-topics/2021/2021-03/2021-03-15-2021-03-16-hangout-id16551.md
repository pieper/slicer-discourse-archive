## Post #4 by @jcfr (2021-03-16 17:42 UTC)

<h2><a name="p-56434-notes-and-action-items-1" class="anchor" href="#p-56434-notes-and-action-items-1" aria-label="Heading link"></a>Notes and Action Items</h2>
<ul>
<li>
<p>Discussed issue with having more than one CLI running parallel</p>
<ul>
<li>Multiple CLIs can be scheduled for execution but here is only one processing thread</li>
<li>Testing code in <a href="https://github.com/KitwareMedical/SlicerParallelTest">KitwareMedical/SlicerParallelTest</a> demonstrate the current issues</li>
<li>Topic <a href="https://github.com/Slicer/Slicer/compare/master...jcfr:support-running-cli-in-parallel">jcfr/Slicer@support-running-cli-in-parallel</a> updates <code>vtkSlicerApplicationLogic</code> to create multiple processing thread. It is a work-in-progress and requires more testing as there are issues with properly reading stream of processes and updating status of relevant command line module nodes.</li>
</ul>
</li>
<li>
<p>Integrated <a href="https://github.com/Slicer/Slicer/pull/5528">PR5528</a> and <a href="https://github.com/Slicer/Slicer/pull/5527">PR-5527</a></p>
</li>
<li>
<p>libcloud provides an abstraction to interface with google cloud, aws, digital ocean, … see <a href="https://libcloud.apache.org/">https://libcloud.apache.org/</a> . <a class="mention" href="/u/pll_llq">@pll_llq</a> as some experience using it.</p>
</li>
<li>
<p>versioning</p>
<ul>
<li>add updated versioning scheme document to readthedocs based on <a href="https://www.slicer.org/wiki/Documentation/Nightly/Developers/Versioning#Proposal_.233">Propsal 3</a></li>
</ul>
</li>
<li>
<p>new stable release (<a href="https://github.com/Slicer/Slicer/issues/5525">PR-5525</a>)</p>
<ul>
<li>Do not create any more snapshot releases including <code>YYYYMMDD</code>. This caused issues with version comparison (see <a href="https://discourse.slicer.org/t/segmenteditorextraeffects-not-building-for-latest-stable-snapshot/16291/6">here</a>)</li>
<li>Create a new release called <code>4.14</code> and then update master to be <code>4.15</code></li>
</ul>
</li>
<li>
<p>Reviewed <a href="https://github.com/Slicer/Slicer/pull/5349">Pluggable Markups PR-5349</a> from  <a class="mention" href="/u/rafaelpalomar">@RafaelPalomar</a></p>
<ul>
<li>Avoid exporting module libraries.</li>

</ul>
</li>
<li></li>
<li>
<p><a href="http://slicer.org">slicer.org</a>: new design</p>
<ul>
<li>Address remaining <a href="https://github.com/Slicer/slicer.org/pull/16#issuecomment-791049890">issues</a> (e.g consolidate sitemap and top-toolbar, fix style of “Extensions” and “Community” sections, …)</li>
<li>Get feedback from <a class="mention" href="/u/faiza.ahmed">@faiza.ahmed</a></li>
</ul>
</li>
<li>
