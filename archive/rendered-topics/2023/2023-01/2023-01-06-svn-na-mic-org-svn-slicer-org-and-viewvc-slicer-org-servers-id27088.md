</div><h3><a name="p-88514-relationship-between-slicerslicer-slicerslicergitsvnarchive-and-the-github-repository-archives-described-here-7" class="anchor" href="#p-88514-relationship-between-slicerslicer-slicerslicergitsvnarchive-and-the-github-repository-archives-described-here-7" aria-label="Heading link"></a>Relationship between Slicer/Slicer, <code>Slicer/SlicerGitSVNArchive</code> and the GitHub Repository Archives described here ?</h3>
<ul>
<li>
<p>Following the <a href="https://discourse.slicer.org/t/transition-to-github/10358">transition</a> to using GitHub as the only repository for organizing Slicer sources, former Slicer GitHub repository was renamed to <a href="https://github.com/Slicer/SlicerGitSVNArchive">Slicer/SlicerGitSVNArchive</a>. This was done after we filtered and rewrote the commit history to remove large files, fix authorship and update commit messages <sup class="footnote-ref"><a href="#footnote-88514-1" id="footnote-ref-88514-1">[1]</a></sup></p>
</li>
<li>
<p>The content of <a href="https://github.com/NA-MIC/svn.slicer.org-Slicer4/commits/trunk">NA-MIC/svn.slicer.org-Slicer4@trunk</a> matches the content of <a href="https://github.com/Slicer/SlicerGitSVNArchive/commits/master">Slicer/SlicerGitSVNArchive@master</a>. That said, the commit SHA are different because the entire <code>Slicer4</code> project was extracted using <code>git svn clone -s http://svn.slicer.org/Slicer4/</code> and this led to different git commits.</p>
</li>
</ul>
