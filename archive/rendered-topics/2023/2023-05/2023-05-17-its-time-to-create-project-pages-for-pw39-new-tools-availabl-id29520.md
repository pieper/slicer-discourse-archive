# It's time to create Project Pages for PW39 ! New tools available!

## Post #1 by @Sam_Horvath (2023-05-17 22:00 UTC)

<p>We are excited to announce the new procedures for the Project Week website that will streamline the generation of Project Pages for attendees!</p>
<p>Now, using the <a href="https://github.com/NA-MIC/ProjectWeek/issues/new/choose">Project Week GitHub Issue</a> page, you have three options to create your Project Page:</p>
<ol>
<li>
<a href="https://github.com/NA-MIC/ProjectWeek/issues/new?assignees=drouin-simon%2Cpiiq%2Crafaelpalomar%2Csjh26%2Ctkapur&amp;labels=proposal%2Cevent%3APW39_2023_Montreal&amp;projects=&amp;template=proposal.yml&amp;title=Proposal%3A+">Create a Proposal issue</a>: If you have an idea for a project page but are not quite ready to create it yet, you can create a “Proposal” issue.</li>
<li>
<a href="https://github.com/NA-MIC/ProjectWeek/issues/new?assignees=drouin-simon%2Cpiiq%2Crafaelpalomar%2Csjh26%2Ctkapur&amp;labels=project%2Cevent%3APW39_2023_Montreal&amp;projects=&amp;template=project.yml&amp;title=Project%3A+">Create a Project issue</a>: If you are ready to create your page, you can simply create a “Project” issue.  This issue will allow you to fill out a convenient form to provide the necessary details.</li>
<li>
<a href="https://projectweek.na-mic.org/PW39_2023_Montreal/Projects/">Create the project page yourself using the template</a>: If you prefer to create the Project Page yourself, you can still do so by using the provided template and submitting a pull request.</li>
</ol>
<p>We hope these new procedures will help you to maximize your participation in the Project Week.</p>
<p><strong>Next:</strong> After creating a <em>Project</em> issue, we are working on automatically generating a pull request for you, enabling easy review.</p>
<p>Big thanks to <a class="mention" href="/u/jcfr">@jcfr</a> for developing the new workflow!</p>
