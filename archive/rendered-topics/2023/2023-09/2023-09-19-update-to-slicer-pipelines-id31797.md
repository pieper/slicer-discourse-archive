## Post #1 by @Harald_Scheirich (2023-09-19 20:29 UTC)

<p><a href="https://www.kitware.com/introducing-slicerpipelines-a-coding-free-way-to-create-simple-modules-in-3d-slicer/" rel="noopener nofollow ugc">SlicerPipelines</a> are Slicer’s way to create workflows combining functionality from multiple modules together in one single module. The module created by SlicerPipelines can then be run by themselves or used for batch processing via the Pipeline Case Iterator. This article presents the upgraded functionality of this Slicer extension. The previous architecture allowed us to pass an exact return value from one pipeline stage to the next, and this became a limiting factor for the use of this module. By refactoring to allow for the use of <a href="https://slicer.readthedocs.io/en/latest/developer_guide/parameter_nodes/overview.html" rel="noopener nofollow ugc">Parameter Nodes</a> we can now pass any number of parameters and nodes through any number of stages and include these values in the result set.</p>
<h2><a name="p-100708-multiple-data-for-outputs-1" class="anchor" href="#p-100708-multiple-data-for-outputs-1" aria-label="Heading link"></a>Multiple data for outputs</h2>
<p>In the previous iteration the output of a pipeline was limited to a single output node. In this new version of the Slicer extension a pipeline can now output any number of nodes and/or scalar values. This means you can not only pass the main output of your operation through a pipeline i.e. a segmentation, but also pass along and reuse intermediary results like thresholds, window parameters for use in a later stage. See the <a href="https://github.com/KitwareMedical/SlicerPipelines/blob/main/Documentation/PipelineCreator.md" rel="noopener nofollow ugc">Pipeline Creator Documentation</a> for more information on how to create pipelines.</p>

<h2><a name="p-100708-extending-the-pipeline-creator-3" class="anchor" href="#p-100708-extending-the-pipeline-creator-3" aria-label="Heading link"></a>Extending the Pipeline Creator</h2>
<p>Extending the Pipeline Creator has now become easier with the changes presented. If you want to add support for creating new modules with Pipeline Creator to your own modules you only need to write a very simple function that will be the interface between Pipeline Creator and your module. Mark up the function using the <code>@slicerPipeline</code> decorator and your code will automatically be usable inside of Pipeline Creator. There are implementations of pipeline functions from the surface toolbox, the segment editor, and a subset of VTK filters. See the documentation on <a href="https://github.com/KitwareMedical/SlicerPipelines/blob/main/Documentation/PipelineCreator.md#creating-a-pipeline-in-source" rel="noopener nofollow ugc">creating pipelines</a> for more information, or check <a href="https://github.com/KitwareMedical/SlicerPipelines/tree/main/PipelineModules/_PipelineModules" rel="noopener nofollow ugc">the source</a> of our implementations.</p>
